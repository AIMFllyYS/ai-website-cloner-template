# 通用验收 · 诚实分层 + 验收等级

> 两个用途：①动手前向用户声明能做到哪一层（防止过度承诺）②收尾时按等级硬验收（防止"看着差不多"就宣称完成）。

---

## §1 可行性分层（动手前声明）

| 层 | 定义 | 可行性 |
|---|---|---|
| **T0 资产级完整归档** | 服务器暴露的全部静态资源字节级落盘 | ✅ 任何站可达 |
| **T1 可交互离线镜像** | 同一编译产物本地跑通，视觉/交互与线上一致 | ✅ 需正确服务配置（MIME/头/SPA fallback）；hybrid 站为"数据快照版 T1" |
| **T2 源码级重建（原始工程）** | 还原**打包前的原始**源码、着色器、3D 原始工程 | ❌ 编译产物站不可逆、**禁止声称**。手写静态站无此问题（源码即产物） |
| **T2-AI 等价源码重建** | AI 依线上截图/DOM/computed CSS **重新实现**一份现代栈（Next.js/React/TS）源码，视觉&交互对齐 | ✅ 仅编译站、经 cloner（Stage 2）。**诚实标注"等价重建，非原站原始源码"**。手册 `playbook-reconstruction.md` |

**"1:1 复刻" 的两层诚实定义：**
- **① 镜像级 1:1 = T0 + T1**（Stage 1）。编译站到此为止即合格；这是默认承诺层级。
- **② 源码级等价 = T2-AI**（Stage 2，仅编译站、用户确认后 opt-in）。产出可读可二次开发的等价源码。

任何对编译产物站声称 **T2（原始工程）** 的说法都是误导，技能内禁止；T2-AI **必须**标注为"AI 等价重建，非原站原始源码"。

## §2 验收等级（收尾硬门槛）

| 等级 | 标准 | 验证方法 |
|---|---|---|
| **L0 资产完整** | `DONE.list` 覆盖权威清单全量；`MISSING.log` 每条已分诊（真 404/已降级/已补抓）；关键依赖（wasm/worker/字体/解码器）魔数核验通过 | 日志审计 + `file` 抽检 |
| **L1 离线可跑** | 断网、本地起服务、站点正常启动渲染，Console 零致命错误 | 手动浏览器测试 |
| **L2 交互一致** | 全部交互状态（滚动各段/点击切换/音频/路由跳转+刷新）与线上一致；**DevTools Network 零 404、零 soft-200** | 逐状态对照线上 |
| **L3 像素级一致** | 关键状态截图与线上同版本浏览器逐像素对比 | 截图叠加 |
| **L4 字节级一致** | 抽样资源 sha256 与线上一致 | 哈希抽检 |

**各形态的"1:1 完成"判定：**

| 形态 | 必过 | 加分 |
|---|---|---|
| 纯静态/SSG | L0+L1+L2+L3 | L4（CDN 压缩可致字体/图片哈希不同，记已知差异） |
| DOM SPA / SSR | L0+L1+L2 | L3 |
| WebGL 体验站 | L0+L1+L2 | L3（GPU/驱动渲染差异记"已知差异"，不算失败） |
| 叠加 hybrid | 上述 + 录制路径全部离线可达、无逃逸请求 | — |

**Stage 2（T2-AI 等价源码重建）验收门**（仅编译站、走完 Stage 1 后）：

| 项 | 标准 |
|---|---|
| 可运行 | `<project-name>-clone/` `npm run dev` 起得来，首屏正常渲染 |
| 工程健康 | `npm run check`（lint+typecheck+build）**全过** |
| 视觉对齐 | 逐区块对照 `<project-name>-mirror/`（或 live），关键区块对齐 |
| 资产复用 | 图/视频/字体取自 mirror，未重新打服务器 |
| 诚实标注 | 交付含 `playbook-reconstruction.md` §5 声明，标注 **T2-AI 等价重建 ≠ 原始源码** |

未过任一项不得宣称 Stage 2 完成；偏差大→复跑/修，仍偏→降级回交 Stage-1 镜像并说明未达标。

## §3 验收操作清单

1. 断网（或 DevTools offline）。
2. 起对应服务器（静态：任意；SPA：`--spa`；WebGL：serve.js 全 MIME+COOP/COEP）。
3. DevTools Network 全程开着，**逐状态**走完整个站（每个路由、每段滚动、每个交互）。
4. 过滤器查两类失败：状态 404；**类型为 html 但请求的是资源**（soft-200 的镜像残留）。
5. Console 零致命错误（警告逐条判级）。
6. 与线上并排对照关键状态截图。

## §4 交付报告模板

```
## 复刻交付报告 <站点> <抓取时间>
- 形态/手册：<static|spa|webgl>(+hybrid)　承诺层级：T0+T1<+T2-AI 若做了 Stage 2>
- 资产：DONE <N> 条 | MISSING <M> 条（分诊：真404 a / 降级占位 b / 补抓成功 c）
- 验收：L0 ✓ L1 ✓ L2 ✓ L3 <✓/部分> L4 <抽检结果>
- Stage 2（如有）：<project>-clone 可跑 ✓ | npm run check ✓ | 视觉逐区对齐 <✓/部分>　**标注：AI 等价重建，非原站原始源码**
- 已知差异：<GPU渲染/CDN压缩/水合警告/数据快照时间…>
- 不可复刻项：<T2 原始源码（字节级原始工程） / 登录态 / 实时数据…>
- 合规：仅本地/研究用途；公网部署需权利方授权
```

每个 Phase 过程中的一行汇报格式：`Phase N ✓ <完成项> | 资源 X 个 | MISSING Y 条 | 下一步 <...>`
