# 手册 D · 依赖后端 API 的站（叠加手册）

> 适用：页面运行时向 `/api/`、GraphQL、第三方数据源发请求才有内容。**本手册总是与 A/B/C 之一叠加使用**（前端壳按对应手册抓，数据层按本手册处理）。
> 可达上限：**T1 降级版**——"抓取时刻的数据快照 + 本地回放"。动手前必须向用户声明：复刻的是**某一时刻的切片**，不是活服务。

---

## 不可复刻项（先声明，禁止隐瞒）

| 类别 | 原因 |
|---|---|
| 登录态 / 个性化内容 | 凭据与会话属于用户和服务端，镜像无意义且涉隐私 |
| 支付 / 表单提交 | 需要真实后端，本地只能 mock 成功响应做演示 |
| 实时数据（行情/聊天/WebSocket） | 只能录一段回放，不是实时 |
| 服务端逻辑（搜索、推荐） | 只能回放录过的查询，新查询无结果 |

## Phase 2-API · 录制数据层

与前端资产 HAR 不同，**API 录制必须带响应体**：

```bash
HAR_CONTENT=embed node scripts/capture.js "$BASE/" HAR-API 30 1000
```

录制时把会产生数据请求的交互都走一遍（翻页、筛选、搜索常用词、打开详情）。

## Phase 3-API · 构建 mock 树

把每个 API 响应按 `路径 + 规范化查询串` 落盘：

```bash
jq -r '.log.entries[] | select(.request.url | test("/api/|graphql"))
  | [.request.url, .response.content.text] | @tsv' HAR-API/*.har
# → 落盘为 mock/api/<path>/<querystring哈希>.json
```

要点：
- 查询参数**排序后再做键**（`?b=2&a=1` 与 `?a=1&b=2` 是同一请求）。
- GraphQL 按 `operationName + variables哈希` 做键。
- 响应里的绝对 URL（图片 CDN 等）也要提取出来，并入前端资产清单去下载。

## Phase 5-API · 本地回放（三选一，按简单优先）

1. **静态 JSON 直出**：API 路径规则简单时，直接把 mock 树按原路径放进镜像目录，`serve.js` 原样伺服（GET-only 站点首选）。
2. **本地代理**：serve.js 前挂一个小代理，命中 mock 键则返回落盘 JSON，未命中返回 404 并记账。
3. **Service Worker 回放**：往 index.html 注入 SW 拦截 fetch——侵入式修改，记入"已知差异"。

POST/带鉴权的请求：mock 一个固定成功响应即可演示流程，**向用户明示这是假数据**。

## 验收补充（叠加在主手册验收之上）

- 录制过的每条数据路径离线可达，页面数据区不空白。
- 未录制的查询触发兜底（404 或空态），**不**逃逸请求线上真 API（断网测试可验证）。
- 交付报告里单列：数据快照时间、已录 API 清单、mock 覆盖率、不可复刻项。
